# AGENTS.md

## 项目名称

业档底座

## 项目背景

这是一个本地 HTML 原型项目，只有前端页面，不包含后端服务，不连接接口，不涉及数据库，也不承担生产部署能力。

本项目用于水务行业档案系统及相关 AI 功能的产品展示和交互原型演示。

主要使用对象包括：产品、测试、开发及其他项目相关人员。

## 项目目标

- 展示水务行业档案系统的核心业务流程。
- 展示上传文件申请与审批流程。
- 展示审批通过后自动生成电子文件台账的业务逻辑。
- 展示档案系统的基础配置标准。
- 展示 AI 辅助档案处理能力及对应规则维护页面。

## 技术范围

- 静态 HTML
- CSS
- JavaScript
- 无后端连接
- 无接口集成
- 无数据库
- 无生产部署假设

## 本地预览

在项目根目录启动本地服务：

```bash
python3 -m http.server 8000 --bind 127.0.0.1
```

浏览器访问：

```text
http://127.0.0.1:8000/
```

## 重要文件

- `index.html`：主页面文件，包含主要页面结构和交互逻辑。
- `style.css`：主样式文件。
- `.gitignore`：Git 忽略规则。
- `AGENTS.md`：面向 Codex/AI 协作者的项目说明与协作规则。

## 核心业务模块

### 1. 工作台

用于展示系统入口、任务概览、业务提醒和档案工作状态等内容。

### 2. 上传文件

用于发起上传申请和审批流程。

这是当前原型中的重点业务流程之一。修改相关内容时，需要保持“发起上传申请 -> 审批流程 -> 审批结果”的演示逻辑清晰完整。

### 3. 电子文件台账

上传申请审批通过后，系统自动生成电子文件台账。

该模块需要清楚体现“审批通过”和“台账生成”之间的业务关系。

### 4. 系统配置

当前档案系统的标准配置包括：

- 档案分类管理
- 档案目录管理

这两个页面代表档案系统的基础配置标准，修改时应保持结构清晰、层级明确、表达稳定。

### 5. AI 功能

当前原型包含 3 个 AI 功能：

- AI 校验
- AI 提取
- AI 分类

相关规则库页面包括：

- AI 校验规则库
- AI 提取规则库

AI 功能含义：

- AI 校验：对上传的档案文件进行合规性、风险性审查校验。
- AI 分类：对上传的档案文件类型进行自动分类。
- AI 提取：对上传文件内容中的关键字段进行提取。

## 重点页面逻辑

后续修改时，优先关注并保护以下关键逻辑：

- 上传文件申请
- 审批流程展示
- 审批通过后生成电子文件台账
- AI 规则维护
- AI 校验、AI 提取、AI 分类的功能表达

修改这些内容时，应避免破坏页面之间的业务连续性和演示叙事。

## 视觉与交互要求

- 页面应适合作为产品展示和交互原型使用。
- 保持现有视觉风格，除非用户明确要求重新设计。
- 页面层级应清晰，便于产品、测试、开发理解。
- 避免无关装饰和过度复杂的交互。
- 不要引入后端、接口、数据库等真实系统假设。
- 交互应服务于演示目的，保持清楚、直观、可解释。
- 涉及视觉或交互改动时，应在本地浏览器中检查效果。

## 代码修改原则

- 修改前先阅读相关 HTML、CSS、JavaScript。
- 优先做小范围、明确目标的修改。
- 不要随意删除已有业务内容。
- 不要引入不必要的框架、构建工具或依赖。
- 保持静态前端原型的简单性和可维护性。
- 修改布局或交互后，需要尽量在浏览器中验证页面效果。

## Codex 协作规则

- 对于简单明确的任务，Codex 可以在理解需求后直接执行。
- 对于复杂或不明确的问题，Codex 应先确认任务目标、范围和预期结果。
- 如果对业务含义、页面流程、视觉方向存在疑问，Codex 应先询问用户，再开始执行。
- Codex 可以在需要时启动本地预览服务查看页面。
- Codex 不应要求用户提供 SSH 私钥、密码或 token。
- 如果需要 GitHub 权限，Codex 应检查本机 SSH/Git 登录状态，而不是向用户索要敏感信息。

## Git 工作流

- 默认分支：`main`
- 远程仓库：`git@github.com:xdqqing/PM-yedang.git`
- 进行较大修改前，可先拉取最新代码。
- 提交前应检查工作区状态。
- 提交信息应清楚说明本次修改内容，可以使用中文。
- 不要自动 push。
- 只有在用户明确要求 push 时，Codex 才可以执行 push。

## 不属于本项目范围

- 接口集成
- 数据库设计
- 生产部署
- 真实鉴权或权限系统
- 真实文件上传处理
